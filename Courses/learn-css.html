01

ما هو CSS؟

التعريف

CSS (Cascading Style Sheets) لغة التنسيق لصفحات HTML. بدونها الصفحة مجرد نص عادي. معها تصبح واجهة جميلة وتفاعلية.

💡
كيف يعمل؟ المتصفح يقرأ HTML أولاً، ثم يطبق CSS. HTML = هيكل المبنى، CSS = الدهان والديكور.
syntax.css
/* selector { property: value; } */

h1 {
  color: #2965f1;
  font-size: 32px;
  font-weight: bold;
  text-align: center;
}
p {
  color: #555;
  line-height: 1.6;
  margin-bottom: 16px;
}
02

طرق إضافة CSS

1. External CSS (الأفضل)

ملف .css منفصل يُربط بـ HTML. الطريقة الاحترافية — تفصل الكود وتتحكم فيه من مكان واحد.

2. Internal CSS

داخل <style> في الـ <head>. مفيد للصفحات الصغيرة أو الاختبار السريع.

3. Inline CSS (تجنبها)

على العنصر مباشرة style="...". أعلى أولوية لكن تصعّب الصيانة والتنظيم.

4. CSS Modules (React)

ملفات .module.css. كل كلاس scoped للكومبوننت، يمنع التعارض.

index.html
<!-- 1. External -->
<link rel="stylesheet" href="style.css">

<!-- 2. Internal -->
<style> body { background: #fff; } </style>

<!-- 3. Inline -->
<h1 style="color:red; font-size:24px">عنوان</h1>

المحددات Selectors

ما هو المحدد؟

يخبر CSS على أي عنصر HTML يطبّق التنسيق. من أبسط أشكاله (اسم العنصر) لأعقده (محددات مركبة وشروط).

*
Universal — يختار كل العناصر
div
Type — يختار نوع العنصر
.class
Class — يختار بالـ class
#id
ID — فريد في الصفحة
div p
Descendant — كل p داخل div
div > p
Child — الابن المباشر فقط
h1 + p
Adjacent — p بعد h1 مباشرة
h1 ~ p
Sibling — كل p بعد h1
a[href]
Attribute — عنصر بخاصية
a[href*="x"]
Contains — يحتوي على نص
:hover
Pseudo-class — حالة التحويم
::before
Pseudo-element — عنصر افتراضي

a[href$=".pdf"]     { color: red; }   /* ينتهي بـ */

.item::before       { content: '★ '; color: gold; }

Responsive Design

Mobile First — الأساس الصحيح

ابدأ بالموبايل أولاً، ثم أضف @media للشاشات الأكبر. يضمن أداءً أفضل ويسهّل الكود.

الصفحة على شاشات مختلفة
📱 Mobile <768px
💻 Tablet <1024px
🖥️ Desktop >1024px
responsive.css
/* Mobile First — الكود الأساسي = موبايل */
.grid { display: grid; grid-template-columns: 1fr; }

@media (min-width: 768px)  { .grid { grid-template-columns: repeat(2,1fr); } }
@media (min-width: 1024px) { .grid { grid-template-columns: repeat(3,1fr); } }

/* Dark Mode */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0a0e1a; --text: #e8eaf6; }
}
/* وحدات مرنة */
.fluid  { font-size: clamp(16px, 2.5vw, 24px); }
.box    { width: min(100%, 600px); }
/* px=ثابت | em=نسبي للأب | rem=للـroot | vw/vh=الشاشة | dvh=dynamic */

Specificity الأولوية

ليه CSS بتتجاهل كودي؟

لما تتعارض قاعدتين، CSS يختار الأكثر "أولوية". فهم ده يحل 90% من مشاكل "ليه مش شغال!"

مقياس الأولوية من الأقوى للأضعف
!important
أعلى — تجاوز كل شيء (تجنب!)
style="" inline
1000 نقطة
#id
100 نقطة
.class / :pseudo / [attr]
10 نقاط
element (div, p)
1 نقطة
specificity.css
p { color: red !important; }    /* ⚠️ استخدم بحذر */
#header p        { color: green; }   /* 101 = 100+1 */
.text.highlight  { color: blue; }    /* 20 = 10+10 */
.container p     { color: orange; }  /* 11 = 10+1 */
p                { color: black; }   /* 1 */
/* عند التساوي — الأخير يفوز */
p { color: red; }
p { color: blue; }   /* يفوز */
/* الوراثة — color و font-family تورّثان تلقائياً */
/* margin و padding و border لا تورث */

Modern CSS 2024+

مميزات CSS الحديثة

هذه المميزات كانت تحتاج JavaScript أو مكتبات. الآن موجودة في CSS وحدها ومدعومة في كل المتصفحات.

modern-css.css
/* CSS Nesting (2023+) — مثل Sass */
.card {
  background: white;
  & h2    { color: #2965f1; }
  &:hover { transform: scale(1.02); }
  @media (max-width: 768px) { padding: 12px; }
}

/* :has() — Parent Selector (2023+) */
.form:has(input:invalid)  { border-color: red; }
section:has(img)           { display: grid; }

/* @layer — تنظيم أولوية الكود */
@layer base, components, utilities;
@layer base      { h1 { font-size: 2rem; } }
@layer utilities { .blue { color: #2965f1; } }

/* Container Queries — نسبي للـcontainer */
.sidebar { container-type: inline-size; }
@container (min-width: 300px) {
  .card { display: flex; }
}

/* Scroll-driven Animations */
.section {
  animation: fadeIn linear;
  animation-timeline: view();
  animation-range: entry 0% cover 20%;
}

/* :is() و :where() */
:is(h1, h2, h3)    { font-weight: 700; }  /* بأولوية عادية */
:where(h1, h2, h3) { font-weight: 700; }  /* بأولوية صفر */

/* oklch — الأدق في الألوان */
:root { --primary: oklch(60% 0.2 250); }

/* View Transitions */
::view-transition-old(root) { animation: fade-out 0.3s; }
::view-transition-new(root) { animation: fade-in  0.3s; }

التصميم المتجاوب المتقدم Advanced Responsive Design

ما معنى Responsive Design فعلاً؟

التصميم المتجاوب ليس مجرد "الموقع يظهر على الموبايل". هو فلسفة تصميم كاملة تضمن أن الموقع يقدم أفضل تجربة مستخدم على أي جهاز — موبايل، تابلت، لابتوب، تلفزيون، ساعة ذكية. المفهوم ظهر مع Ethan Marcotte عام 2010، وأصبح معياراً أساسياً في تطوير الويب.

1. Fluid Grids

شبكات بنسب مرنة (fr, %) بدلاً من قيم ثابتة (px). تتوسع وتتقلص مع حجم الشاشة.

2. Flexible Images

صور تتكيف مع حجم حاويتها. لا تتجاوز عرضها. max-width: 100%

3. Media Queries

قواعد CSS تُطبَّق فقط عند تحقق شرط معين كحجم الشاشة أو توجهها.

// نقاط التكسر Breakpoints

اختيار نقاط التكسر الصحيحة

الخطأ الشائع هو استخدام breakpoints ثابتة مبنية على أجهزة محددة. الأصح هو إضافة breakpoint عندما يبدأ التصميم يبدو كسيحاً. ومع ذلك، الأرقام الشائعة حالياً هي: 320px (موبايل صغير)، 480px، 768px (تابلت)، 1024px (لابتوب)، 1280px، 1440px (سطح المكتب).

breakpoints.css — نظام كامل
/* ===== نظام Breakpoints احترافي ===== */
/* xs: أصغر من 480px */
/* sm: 480px+ */
/* md: 768px+ */
/* lg: 1024px+ */
/* xl: 1280px+ */
/* 2xl: 1536px+ */

/* الكود الأساسي للموبايل (xs) */
.container {
  width: 100%;
  padding: 0 16px;
  margin: 0 auto;
}

/* sm: 480px+ */
@media (min-width: 480px) {
  .container { padding: 0 24px; }
}

/* md: 768px+ */
@media (min-width: 768px) {
  .container { max-width: 768px; padding: 0 32px; }
}

/* lg: 1024px+ */
@media (min-width: 1024px) {
  .container { max-width: 1024px; }
}

/* xl: 1280px+ */
@media (min-width: 1280px) {
  .container { max-width: 1280px; }
}

/* ===== تعريف المتغيرات في :root ===== */
:root {
  --bp-sm:  480px;
  --bp-md:  768px;
  --bp-lg:  1024px;
  --bp-xl:  1280px;
}
// وحدات Viewport
viewport-units.css — الوحدات المرنة بالتفصيل
/* ===== وحدات الـ Viewport ===== */

/* vw = viewport width (عرض الشاشة) */
.hero { width: 100vw; font-size: 5vw; }

/* vh = viewport height (ارتفاع الشاشة) */
.full-screen { height: 100vh; }

/* ⚠️ مشكلة vh في الموبايل — الشريط العلوي للمتصفح */
/* الحل: dvh (Dynamic Viewport Height) */
.hero-mobile {
  height: 100dvh;   /* يأخذ في الاعتبار شريط المتصفح */
}

/* svh = Small Viewport Height (أصغر viewport محتمل) */
/* lvh = Large Viewport Height  (أكبر viewport محتمل) */
/* dvh = Dynamic Viewport Height (يتغير مع الشريط) */

/* vmin = الأصغر من vw أو vh */
/* vmax = الأكبر من vw أو vh */
.circle { width: 50vmin; height: 50vmin; }

/* cqw = Container Query Width (حديث!) */

/* clamp(min, preferred, max) */
h1 { font-size: clamp(28px, 5vw, 64px); }
h2 { font-size: clamp(22px, 3.5vw, 40px); }
h3 { font-size: clamp(18px, 2.5vw, 28px); }
p  { font-size: clamp(14px, 1.5vw, 18px); }

/* الصيغة الرياضية المتقدمة */
/* font-size = min + (max - min) * (vw - min-vw) / (max-vw - min-vw) */
h1 {
  /* من 24px عند 320px شاشة حتى 64px عند 1440px شاشة */
  font-size: clamp(1.5rem, calc(1.5rem + 2.5vw), 4rem);
}

// Container Queries — الثورة الحقيقية

ليه Container Queries أهم من Media Queries؟

Media Queries تعتمد على حجم الشاشة الكلي. Container Queries تعتمد على حجم الحاوية الأب مباشرة. فمثلاً كارد في سايدبار ضيق يتصرف بشكل مختلف عن نفس الكارد في المحتوى الرئيسي، رغم أن حجم الشاشة نفسه.

container-queries.css — المستقبل هنا
/* ===== Container Queries ===== */

/* 1. تعريف الـ Container */
.card-wrapper {
  container-type: inline-size;  /* يراقب العرض فقط */
  container-name: card;         /* اسم اختياري */
}
/* اختصار */
.wrapper { container: card / inline-size; }

/* 2. استخدام @container */
@container (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
  .card-img { width: 200px; }
}

@container card (min-width: 600px) { /* باسم محدد */
  .card-title { font-size: 24px; }
}

/* 3. وحدات الـ Container */
.responsive-text {
  font-size: 5cqi;   /* Container Query Inline (عرض) */
  padding: 2cqb;    /* Container Query Block (ارتفاع) */
  margin: 1cqw;     /* Container Query Width */
}

/* 4. Container Query بدون اسم */
@container style(--variant: primary) { /* Style Queries (جديد!) */
  .btn { background: #2965f1; }
}

✅ افعل دائماً

• ابدأ من الموبايل (Mobile First)
• استخدم وحدات مرنة: rem, em, %, vw
• اختبر على أجهزة حقيقية
• استخدم max-width: 100% للصور
• استخدم clamp() للحجوم المرنة
• اضبط meta viewport
• فكر في المحتوى أولاً

❌ تجنب دائماً

• قيم px ثابتة للعرض الكامل
• Overflow مخفي يكسر التصميم
• نص صغير جداً على الموبايل
• عناصر قابلة للنقر أصغر من 44px
• افتراض أن المستخدم بفأرة
• Breakpoints مبنية على أجهزة محددة
• تجاهل اتجاه النص RTL/LTR
